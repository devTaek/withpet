# Withpet
#### 반려견의 일상을 기록하고, 주변 산책로 및 동물병원 정보를 확인하며, 반려견 정보를 관리할 수 있는 커뮤니티 기반 플랫폼입니다.

### ✅주요기능

[마이페이지]
<img width="1727" alt="스크린샷 2025-04-11 오후 4 00 39" src="https://github.com/user-attachments/assets/7f1eb29e-379e-4121-a615-bb2ae009b5eb" />
  1. Redux 상태관리 및 LocalStorage 이용
    - 로그인 성공 시, 서버로부터 받은 사용자 정보(user) 및 반려동물 정보(pet)를 Redux 전역 상태에 저장하고, 동시에 localStorage와 동기화하여 유지됩니다.
    - 웹페이지를 새로고침하더라도 localStorage.getItem()을 통해 지속적으로 접근할 수 있습니다.
  2. 로그인 요청 및 JWT 인증 처리
    - JWT 토큰을 HttpOnly 옵션이 적용된 쿠키에 저장하여 클라이언트에 전달합니다.
    - JWT 인증에 대한 authAxios(전처리 & 후처리) 구현하여, 토큰 유효성 체크 및 오류 처리가 자동으로 이루어지도록 하였습니다.


[반려견 피드]

<img width="1727" alt="스크린샷 2025-04-11 오후 4 22 43" src="https://github.com/user-attachments/assets/9e327e92-01af-4b60-a4f3-185e015fd09e" />

  1. 데이터베이스 테이블 구조 및 관계 설정
    사용자(user) 테이블의 Primary Key를 기준으로, 다음과 같은 테이블과 외래 키(Foreign Key)를 통해 관계를 구성합니다.
      - feed: 사용자가 작성한 게시글 정보 저장(기반 테이블)
      - img: 피드에 첨부된 이미지 저장
      - comment: 피드에 대한 사용자 댓글 저장
      - like: 피드 또는 댓글에 대한 좋아요 기록 저장
  
  2. 실시간 댓글 시스템 구현 – Socket.IO 활용
      - 클라이언트와 서버 간 양방향 실시간 통신을 구현했으며, 댓글 조회, 작성, 삭제 기능을 이벤트 기반 구조로 처리합니다.
      - 모든 변경사항은 io.emit을 통해 전체 사용자에게 브로드캐스트되어 실시간으로 UI가 동기화됩니다.
      - 이를 통해 사용자는 페이지 새로고침 없이도 실시간으로 댓글을 확인할 수 있음


[산책로 및 동물병원 검색]

<img width="1727" alt="스크린샷 2025-04-11 오후 4 09 52" src="https://github.com/user-attachments/assets/d4c7a33c-fd7f-43dc-bcc1-90e69330d01b" />

  1. 산책로 데이터: 서울시 공공데이터 API 활용
    - 서울시에서 제공하는 공공 API를 통해 산책로 데이터를 수집합니다.
    - 해당 API는 산책로의 지리 좌표(위도, 경도) 및 명칭, 거리 등의 메타데이터를 포함하고 있어, 이를 기반으로 지도에 마커를 표시했습니다.
    - 사용자의 현재 위치를 기준으로 반경 필터링 또는 거리 계산을 통해 가까운 산책로를 추천하는 기능도 구현 가능합니다.

  2. 동물병원 정보: 카카오맵 API 검색 기능 사용
    - 카카오맵의 장소 검색 API를 통해 “동물병원” 키워드로 주변 병원을 검색합니다.
    - 검색 결과는 카카오 지도 위에 마커 형태로 시각화하여 사용자에게 직관적인 정보 제공이 가능합니다.
    - 장소 클릭 시 정보창(InfoWindow)을 통해 세부 정보도 출력 가능하게 구성
    - 사용자의 현재 위치를 기준으로 반경 필터링 또는 거리 계산을 통해 가까운 산책로를 추천하는 기능도 구현 가능합니다.
